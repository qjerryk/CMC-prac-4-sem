Шахматы
Написать полную программу,(вместе с main) проверяющую соответствие хода шахматным правилам.

Шахматная доска это поле из 8*8=64 клеток, по горизонтали обозначаются буквами от a до h, а по вертикали числами от 1 до 8. Поэтому любая клетка задается парой буква-цифра. Например, e4.

Запрос пользователя имеет вид:

[код фигуры] [начальная клетка] [целевая клетка]

Коды фигур: K - король, Q - ферзь, N - конь, B - слон, R - ладья (пешек нет)

Программа должна ответить на вопрос - может ли данная фигура пойти с указанной начальной клетки на целевую клетку.

Программа должна состоять из следующих классов: ChessChecker, Parser, абстрактный класс Figure и унаследованные от Figure классы, описывающие шахматные фигуры (король, ферзь и т.д.).

Класс ChessChecker - основной, он принимает запрос от пользователя, то есть у него есть метод, который считывает входные данные. Дальше полученные данные передаются методам класс Parser

Класс Parser необходим для разбора запроса пользователя, перевода его в удобный для ChessChecker вид. Вместо Parser допустимо использовать другое решение для преобразования запроса пользователя в вид удобный для проверки законности хода.

Фигуры храняться внутри класса ChessChecker в виде массива указателей на Figure В классе Figure реализован чисто виртуальный метод move, который переопределяется в каждом классе-наследнике в соответствии с правилами шахмат.

После преобразования в удобный вид осуществляет проверка законности хода с помощью класса, описывающего соответствующую фигуру (вызывая метод move).

Ходы фигур выполнять по следующим правилам:

король — передвигается со своего поля на одно из свободных смежных полей
ферзь — ходит по вертикалям, диагоналям и горизонталям, на которых она находится.
ладья — ходит по вертикалям и горизонталям, на которых она находится.
слон — ходит по диагоналям, на которых он находится.
конь — может пойти на одно из полей, ближайших к тому, на котором он стоит, но не на той же самой горизонтали, вертикали или диагонали.
Ходить за пределы доски нельзя!

Input format
[код фигуры] [начальная клетка] [целевая клетка]
Output format
YES or NO
